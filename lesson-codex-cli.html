Codex CLI: un agente de código que actúa, no solo habla

Codex CLI es una herramienta de terminal de OpenAI (open source, MIT) que usa GPT-4o u o3 para leer tu proyecto, editar archivos y correr comandos — de forma autónoma.

Metáfora: Imaginate contratar a un desarrollador junior que no duerme. Le escribís una instrucción en la terminal, él lee todo tu código, hace los cambios y te muestra el resultado. Si no te gusta, lo revertís. Si sí, seguís adelante.
Lección · Slide 3 de 5

¿Cómo lo usás?

Tres modos según cuánta autonomía querés darle:

💬 Suggest Propone cambios. Vos aprobás cada uno antes de que toque algo.
✏️ Auto-edit Edita archivos directamente sin preguntar. Comandos de terminal sí pide permiso.
⚡ Full-auto Hace todo solo — edita y ejecuta. Usarlo en sandbox para evitar sorpresas.
  • Instalación en segundos: npm install -g @openai/codex
  • Código 100% abierto (MIT) — podés leer exactamente qué hace

En la práctica

Ejemplo real

Situación: Martín tiene un proyecto con 20 archivos de JavaScript. Necesita agregar manejo de errores a todas las llamadas a la API externa — tarea repetitiva y lenta.


Con Codex CLI: escribe en la terminal:
codex --approval-mode auto-edit "add try/catch error handling to all API calls"


Resultado: El agente lee los 20 archivos, identifica cada llamada a la API y agrega el manejo de errores. Martín revisa el diff y aprueba. Lo que le hubiera tomado 2 horas: 4 minutos.
