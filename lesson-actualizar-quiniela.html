Terminó un partido. ¿Cómo sabe la quiniela el resultado?

La quiniela no se actualiza sola. Alguien tiene que escribir el resultado — como un árbitro que anota en una planilla oficial.

Ese "alguien" sos vos. El archivo que actualizás se llama results.js. Un cambio, guardar, subir — y toda la familia ve el leaderboard actualizado.

📋 results.js — la planilla de resultados

results.js es el único archivo que tocás para actualizar la quiniela. Tiene una línea por partido. Cuando cargás el resultado y lo subís a GitHub, la app se actualiza sola para todos.

Metáfora: La quiniela es un marcador electrónico de estadio. results.js es el control remoto. Vos apretás los números, el marcador los muestra para todo el público. Sin el control, el marcador queda congelado en el último resultado.

⚙️ Los 4 pasos para cargar un resultado

1
Abrís results.js con VS Code o cualquier editor de texto
2
Buscás la línea del partido — cada uno tiene un código como m09, m14, etc. con el nombre del partido en comentario al lado
3
Cambiás null por [goles_local, goles_visitante] — ejemplo: [3, 1] — y guardás el archivo
4
Corrés los 3 comandos de git para subir el cambio a GitHub → la familia recarga el browser y ve el leaderboard actualizado
Lección · Slide 4 de 5

🗂️ Ejemplo: Alemania venció a Curazao 4 a 0

Qué cambiás en results.js
❌ Antes (sin resultado) "m09": null, // Alemania vs Curazao
✅ Después (con resultado) "m09": [4, 0], // Alemania vs Curazao
Luego en la terminal, los 3 comandos de siempre:

git add results.js
git commit -m "Resultados 13-Jun"
git push
En 1 minuto, boadaedgar.github.io/quiniela-familia muestra los puntos actualizados para todos.
