Figma API

MCP = Model Context Protocol. It is a standard way for Claude to call external tools — like a USB port for AI integrations. Figma publishes an MCP server that exposes their API as Claude-callable tools.

Key insight: Claude never opens Figma in a browser. It calls the Figma REST API through MCP tools, invisibly, in the background.
Section 1 — The Stack

What is a brandbook in this context?

A DESIGN.md file that defines colors, typography, spacing, and component rules in plain text. Claude reads it before writing any HTML.

# DESIGN.md — ayayai brandbook colors: accent: "#f5a623" # amber — primary brand color bg: "#0f0f1a" # deepest dark background surface: "#1a1a2e" # card / panel background blue: "#4fc3f7" # secondary accent green: "#81c784" # success / win red: "#e57373" # error / alert
Why this matters: Claude translates these tokens directly into CSS variables — brand consistency without a designer in the loop.

Section 3 — MCP Tools

The mandatory skill pattern

Before calling create_new_file, Claude must load a skill from the Figma MCP resource. This is a contract — skip it and the tool may behave unexpectedly.

# Step A: load the skill ReadMcpResourceTool( server = "figma", uri = "skill://figma/figma-create-new-file/SKILL.md" ) # Step B: now call create_new_file safely mcp__figma__create_new_file( fileName = "Nota Quiniela Server", planKey = "team::1647650559785762643", editorType = "design" )
Why: The skill encodes the plan-resolution contract and post-creation handoff rules. Without it, Claude may skip required steps.
Section 3 — MCP Tools

generate_figma_design — two-phase call

This tool works in two phases. First call returns a captureId. Second call (with captureId) polls until completed.

# Phase 1: get captureId generate_figma_design(fileKey = "5Hr8rCeHsuQD8AQtqa1FGq") → captureId: "ae3a64df-c930-42ad-b440-b2d4c1d53060" # Phase 2: open this URL in browser (local capture) http://localhost:3333/nota-server.html #figmacapture=ae3a64df... &figmaendpoint=https://mcp.figma.com/mcp/capture/.../submit &figmadelay=1000 # Phase 3: poll until completed generate_figma_design( fileKey = "5Hr8rCeHsuQD8AQtqa1FGq", captureId = "ae3a64df-c930-42ad-b440-b2d4c1d53060" )
Each captureId is single-use. Never reuse one for a second capture.

Section 4 — Workflow

The capture flow in detail

The capture script is injected into your HTML. When you open the hash URL, the script fires and sends your page's DOM to Figma's servers, which convert it to a Figma frame.

What happens in the browser

  • Page loads normally
  • Hash params are parsed by capture.js
  • Page DOM is serialized + sent to Figma
  • A toolbar appears confirming capture

What happens in Figma

  • HTML elements → Figma frames
  • CSS styles → Figma fill/stroke/text
  • New page added to your file
  • Frame appears at node-id 1:2
file:// URLs don't work. You must serve via localhost — that's why the HTTP server step is mandatory.

Section 5 — Gotchas & Tips

Pro tips for junior devs

Design tips

  • Always build HTML first, iterate fast
  • CSS variables = brandbook tokens — name them the same
  • Design for fixed width (560px) for WhatsApp cards
  • Leave capture script in HTML — toolbar lets you recapture after edits

MCP tips

  • Run whoami first — always know your planKey
  • Load skills before tools that require them
  • Poll generate_figma_design — don't call once and give up
  • fileKey is required on EVERY poll call, not just the first
Best practice: Keep edits in HTML, recapture to Figma. HTML is your source of truth — Figma is just the export layer.
