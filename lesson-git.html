🌿 ¿Qué es Git?

Git es un sistema que guarda el historial completo de cambios de tus archivos — como una máquina del tiempo para tu trabajo. Podés volver a cualquier versión anterior en cualquier momento.

Metáfora: Imaginá que cada vez que editás un documento, en vez de "Guardar" que sobreescribe todo, tenés un "Guardar versión con descripción". Git hace eso con todos tus archivos — guarda quién cambió qué, cuándo, y por qué. Y podés volver a cualquier foto anterior con un clic.

⚙️ Tres palabras que necesitás conocer

commit Una foto guardada del estado de tus archivos en un momento. Cada commit tiene un mensaje: "agregué el botón actualizar".
repositorio La carpeta de tu proyecto que contiene los archivos + todo el historial de commits. Vive en tu PC o en GitHub.
branch Una línea paralela donde probás cambios sin tocar la versión principal. Si funciona, la unís. Si no, la descartás.

🗂️ Git en el proyecto quiniela

Cómo lo usaríamos Cada vez que avanzamos en la quiniela haríamos un commit:

✓ "agregué resultados en vivo desde la API"
✓ "calculé puntos por apuesta"
✓ "diseñé el leaderboard familiar"

Si algo se rompe → volvemos al commit anterior en segundos.
Si queremos probar algo nuevo → creamos una rama, probamos, y si funciona lo unimos.

GitHub es donde subimos ese historial para que sea visible, compartible y tenga respaldo en la nube.
